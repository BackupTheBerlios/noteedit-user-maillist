> I joint the 2 files, .not and .ly

I looked at them and found out that you're not using the latest
version of NoteEdit (<2.7.4). There are many LilyPond export
improvements made in 2.7.4 and 2.8.0, so I advice you to update to the
latest version of NE. I opened the .not file and fix some spacings
(using underspace/overspace so the lyrics doesn't hit the staff) and
simply export it to LilyPond. And then:

matevz at mojazver:~/tmp$ convert-ly --edit Anonyme_jaiunmari_4vx.ly
/usr/bin/convert-ly (GNU LilyPond) 2.4.5
Processing `Anonyme_jaiunmari_4vx.ly' ... Applying conversions:

matevz at mojazver:~/tmp$ lilypond Anonyme_jaiunmari_4vx.ly
GNU LilyPond 2.4.5
Processing `Anonyme_jaiunmari_4vx.ly'
Parsing...
Interpreting music... [8][16][24]
Preprocessing graphical objects...
Calculating line breaks... [3][6][9][12][15][18][21]
Layout output to `Anonyme_jaiunmari_4vx.tex'...

matevz at mojazver:~/tmp$

It worked out of the box. I'm attaching the results.

About your problem, I think there's something with your LilyPond
installation messed up. Did you install libguile library (or
everything related to guile) beside the LilyPond? If that doesn't
help, contact the user mailing list at LilyPond
(lilypond-user at gnu.org) and the maintainer for Mandrake.

For the version of LilyPond, you should simply run lilypond --version
and it should print the LilyPond version out.
